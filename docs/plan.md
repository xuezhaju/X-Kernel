## 二、第一阶段：调试基础设施

**目标**：让你能方便地输出调试信息。

| 步骤 | 学习内容 | 关键文献/概念 | 自查问题 |
| :--- | :--- | :--- | :--- |
| 1.1 | 封装 UART 驱动 | NS16550A 数据手册、MMIO | UART 有哪些寄存器？LSR 的 bit5 和 bit6 有什么区别？ |
| 1.2 | 实现 `putchar` 和 `puts` | 循环、指针操作 | 为什么要用 `volatile`？不加会怎样？ |

## 九、遇到问题时怎么查

| 问题类型 | 优先查阅 | 备选 |
| :--- | :--- | :--- |
| 指令/寄存器不清楚 | RISC-V 官方规范 | xv6-riscv 源码 |
| 中断/异常不工作 | 特权规范 + QEMU 日志 | rCore 代码 |
| 页表不生效 | Sv39 规范 + QEMU monitor | xv6 的 `vm.c` |
| 调度器不切换 | xv6 的 `proc.c` | 经典 OS 教材 |
| 系统调用不返回 | 特权规范 + 你的 `sret` 代码 | GDB 调试 |

**调试技巧**：
- QEMU 的 `-d int` 可以打印中断日志
- QEMU 的 `info registers` 查看寄存器
- QEMU 的 `info tlb` 查看 TLB

## 十、建议的阅读顺序

**第一周**：读 xv6-riscv 的 `kernel/uart.c`、`kernel/printf.c`，理解驱动和输出  
**第二周**：读 xv6-riscv 的 `kernel/trap.c`、`kernel/plic.c`，理解中断  
**第三周**：读 xv6-riscv 的 `kernel/kalloc.c`、`kernel/vm.c`，理解内存管理  
**第四周**：读 xv6-riscv 的 `kernel/proc.c`、`kernel/swtch.S`，理解进程切换  
**第五周**：读 xv6-riscv 的 `kernel/syscall.c`、`kernel/sysproc.c`，理解系统调用  

每读完一个模块，回到你的内核实现相同功能，不要复制粘贴。
