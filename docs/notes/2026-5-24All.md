# X-Kernel 学习笔记：从零开始写 RISC-V 内核

### 记录时间：2026-05-24

---

## 一、基础概念篇

### Q1: 内核是什么？

**A**: 内核是操作系统最核心的部分，直接管理硬件（CPU、内存、设备），并为应用程序提供系统调用接口。可以理解为“软件和硬件之间的翻译官 + 资源大管家”。

**三个核心功能**：
1. 硬件初始化（设置栈、中断、时钟）
2. 资源管理（分配 CPU 时间、内存空间）
3. 提供系统调用（让应用程序请求内核服务）

---

### Q2: 寄存器是什么？

**A**: 寄存器是 CPU 内部的高速临时存储位置，RISC-V 有 32 个通用寄存器（x0-x31）。

**常用别名**：

| 编号 | 别名 | 用途 |
| :--- | :--- | :--- |
| x0 | zero | 永远为 0 |
| x1 | ra | 函数返回地址 |
| x2 | sp | 栈指针 |
| x10-x17 | a0-a7 | 函数参数/返回值 |
| x5-x7, x28-x31 | t0-t6 | 临时寄存器 |
| x8-x9, x18-x27 | s0-s11 | 保存寄存器 |

---

### Q3: RISC-V 有多少条指令？

**A**: 基础整数指令集（RV32I）有 **40 条**。加上扩展指令集（M/A/F/D/C）可以超过 200 条。RISC-V 采用模块化设计，可按需选择。

---

### Q4: 指令存在寄存器里吗？

**A**: **不！** 指令存在**内存**里，CPU 一条条取来执行。寄存器只存**数据**。

---

## 二、汇编指令篇

### Q5: `addi x9, x9, 10` 需要申请变量吗？

**A**: 在汇编层面，不需要。你直接操作物理寄存器 x9，没有“变量”这个概念。在 C 语言层面，编译器会帮你把变量映射到寄存器。

---

### Q6: `li` 是什么？

**A**: `li` = Load Immediate，加载立即数（伪指令）。例如 `li x9, 123` 把 123 写入 x9。汇编器会自动展开成 `addi x9, x0, 123`。

---

### Q7: `beq` 是什么？

**A**: `beq` = Branch if EQual，相等时跳转。例如 `beq a0, a1, label`，如果 a0 == a1 就跳转到 label。

---

### Q8: 按位与（AND）有什么用？

**A**: 用于**检查特定位是否为 1**、**清零某些位**、**提取位值**。例如 `x & 0x20` 检查第 5 位是否为 1。

**记忆口诀**：
- AND：两个都是 1，结果才是 1（严格）
- OR：有一个是 1，结果就是 1（宽容）
- XOR：两个不一样，结果就是 1（找不同）

---

## 三、内存与栈篇

### Q9: 内存地址的高地址和低地址怎么区分？

**A**: 数字大的叫高地址（在上面），数字小的叫低地址（在下面）。

```
高地址 0x80202000  ┌─────────────┐
                   │             │
                   │   栈        │
                   │             │
低地址 0x80200000  └─────────────┘
```

---

### Q10: 栈是什么？为什么 C 语言需要栈？

**A**: 栈是内存中一块“后进先出”的区域，用于存放：
- 函数局部变量
- 函数返回地址
- 函数参数

C 语言需要栈来支持函数调用和局部变量。没有栈，C 语言无法运行。

---

### Q11: 栈的方向是什么？sp 是什么？

**A**: 
- 栈**向下增长**（从高地址向低地址，地址值减小）
- **sp**（Stack Pointer）是栈指针，指向当前栈顶
- **`_stack_top`** 是栈区域的最高地址边界，编译时固定
- sp 初始指向 `_stack_top`，然后向下移动，**永远不会超过它**

```
高地址 0x80202000  ← _stack_top（栈顶边界）
                   │   栈区域
                   │      ↓ 向下增长
低地址 0x80201000  ← 栈底
```

---

### Q12: 栈溢出会怎样？

**A**: 栈向下增长太猛会撞到下面的 `.bss`/`.data` 段，破坏全局变量，导致内核行为异常甚至崩溃。

**防止措施**：
- 增加栈大小
- 放置栈哨兵（特殊值检测）
- 增加空白缓冲区

---

## 四、UART 串口篇

### Q13: UART 是什么？

**A**: UART（Universal Asynchronous Receiver/Transmitter）是一个硬件模块，负责串行通信。在内核开发中，它是**最重要的调试输出设备**。

**QEMU virt 平台的 UART 信息**：
- 芯片型号：NS16550A
- 数据寄存器地址：`0x10000000`
- 状态寄存器地址：`0x10000005`

---

### Q14: UART 的工作原理是什么？

**A**: 

1. **发送**：CPU 向数据寄存器（`0x10000000`）写入一个字节
2. **UART 硬件**：自动把这个字节转换成串行信号（起始位 → 8位数据 → 停止位）
3. **QEMU**：接收到串行信号，在虚拟终端显示字符

---

### Q15: THRE 是什么？为什么需要检查它？

**A**: THRE（Transmit Holding Register Empty）是 LSR 寄存器的**第 5 位**（掩码 `0x20`），表示发送 FIFO 是否有空位。

- THRE = 1：FIFO 有空位，可以发送下一个字符
- THRE = 0：FIFO 已满，需要等待

**不检查 THRE 的问题**：写入速度超过 UART 发送速度时，FIFO 会满，新数据会丢失。

---

### Q16: `& 0x20` 是什么意思？是地址加 `0x20` 吗？

**A**: **不是！** `0x20` 是**位掩码**，不是地址偏移。

```c
// 正确理解
char lsr_value = *(char *)0x10000005;  // 读取 LSR 寄存器的值
int thre = (lsr_value & 0x20);         // 提取第 5 位（THRE）

// 0x20 = 二进制 00100000，只保留第 5 位，其他位清零
```

**地址 vs 掩码区分**：

| 概念 | 值 | 作用 |
| :--- | :--- | :--- |
| UART 基地址 | `0x10000000` | 数据寄存器地址 |
| LSR 地址 | `0x10000005` | 状态寄存器地址 |
| THRE 掩码 | `0x20` | 提取第 5 位的工具 |

---

### Q17: THR 和 RBR 为什么是同一个地址？

**A**: 这是硬件设计的**地址复用**技术。同一个地址 `0x10000000`：
- **写入**时 = THR（发送保持寄存器），数据被发送出去
- **读取**时 = RBR（接收缓冲寄存器），返回收到的字符

硬件根据 CPU 的读/写信号自动区分。

---

### Q18: 一个内存地址只能存放 8 bit 吗？

**A**: 在 RISC-V 中，每个内存地址对应 **1 字节（8 bit）**。但 CPU 可以一次性访问连续多个字节（`lb`/`lh`/`lw`/`ld`）。

---

### Q19: 为什么不能用 `printf`？

**A**: `printf` 是标准 C 库函数，依赖操作系统提供系统调用（如 `write`）。你的内核没有操作系统，所以必须**直接操作 UART 硬件**自己实现输出。

---

### Q20: `volatile` 是什么？为什么需要它？

**A**: `volatile` 告诉编译器：**这个变量可能被硬件改变，不要优化掉对它的访问**。

```c
volatile char *uart = (char *)0x10000000;
```

没有 `volatile` 时，编译器可能认为连续多次写入同一个地址是无用的而删除，导致 UART 不工作。

---

## 五、头文件与编译篇

### Q21: `.h` 文件是什么？头文件保护符是什么？

**A**: `.h` 文件是头文件，用来声明函数、宏、类型等，供多个 `.c` 文件共享。

**`#ifndef` 头文件保护符** = "if not defined"，防止同一个头文件被多次包含导致重复定义。

```c
#ifndef UART_H      // 如果 UART_H 没有被定义
#define UART_H      // 就定义它

// 头文件内容...

#endif              // 结束
```

---

### Q22: C 语言在没有标准库的环境下能运行吗？

**A**: **能！** C 语言只需要：
1. 栈指针（sp）已设置
2. 代码段（.text）已加载
3. 入口点正确

标准库只是现成的机器码函数集合，不是 C 语言运行的必要条件。你的内核就是自己造零件的例子。

---

## 六、内核启动与内存布局篇

### Q23: 内核的内存布局是什么样的？

**A**: 

```
高地址 0x88000000  ┌─────────────────────┐ ← 物理内存上限
                   │   空闲内存           │
0x80202000  ├─────────────────────┤ ← _stack_top（栈顶，内核最高地址）
                   │   STACK（4KB）      │
0x80201000  ├─────────────────────┤ ← 栈底
                   │   空白缓冲          │
0x80200400  ├─────────────────────┤
                   │   .bss/.data/.rodata │
0x80200000  ├─────────────────────┤ ← 内核起始地址（最低地址）
                   │   OpenSBI 固件       │
0x80000000  └─────────────────────┘
```

- **内核起始地址**：`0x80200000`（最低地址）
- **内核最高地址**：`_stack_top`（约 `0x80202000`）
- **栈在最顶部**：向下增长时不会撞到代码和数据

---

### Q24: OpenSBI 是什么？

**A**: OpenSBI 是运行在 **M 模式**（最高特权级）的固件，在 CPU 上电后第一个运行。它负责：
1. 初始化硬件（时钟、中断、UART）
2. 加载内核到 `0x80200000`
3. 切换到 S 模式，跳转到内核

**你的内核不需要做的事**：UART 初始化、时钟设置、中断控制器配置（OpenSBI 已做）。

---

### Q25: 内核入口为什么用汇编？

**A**: C 语言需要栈才能运行。设置栈指针（sp）必须用汇编，因为这是 C 语言本身做不到的。

---

## 七、UART 驱动代码框架

```c
// driver/uart.h
#ifndef UART_H
#define UART_H

#define UART_BASE   0x10000000
#define UART_DATA   (UART_BASE + 0x00)
#define UART_LSR    (UART_BASE + 0x05)

#define LSR_DR      0x01    // bit0: 有数据可读
#define LSR_THRE    0x20    // bit5: 可发送

void uart_init(void);
void uart_putchar(char c);
void uart_puts(const char *s);
char uart_getchar(void);

#endif
```

```c
// driver/uart.c
#include "uart.h"

void uart_init(void) {
    // QEMU 中 OpenSBI 已初始化，可为空
}

void uart_putchar(char c) {
    volatile char *lsr = (volatile char *)UART_LSR;
    volatile char *data = (volatile char *)UART_DATA;
    
    // 等待 THRE = 1（FIFO 有空位）
    while ((*lsr & LSR_THRE) == 0) {
        // 等待
    }
    
    *data = c;
    
    // 换行时自动加回车
    if (c == '\n') {
        uart_putchar('\r');
    }
}

void uart_puts(const char *s) {
    while (*s) {
        uart_putchar(*s++);
    }
}

char uart_getchar(void) {
    volatile char *lsr = (volatile char *)UART_LSR;
    volatile char *data = (volatile char *)UART_DATA;
    
    // 等待 DR = 1（有数据可读）
    while ((*lsr & LSR_DR) == 0) {
        // 等待
    }
    
    return *data;
}
```

## 八、关键地址速查表

| 地址 | 内容 | 用途 |
| :--- | :--- | :--- |
| `0x80000000` | OpenSBI 入口 | 固件代码 |
| `0x80200000` | 内核起始地址 | 你的代码开始 |
| `0x80201000` | 栈底 | 栈区域最低地址 |
| `0x80202000` | `_stack_top` | 栈顶，内核最高地址 |
| `0x10000000` | UART 数据寄存器 | 写入=发送，读取=接收 |
| `0x10000005` | UART 线路状态寄存器 | bit5=THRE（可发送），bit0=DR（有数据）|

## 九、核心要点一句话总结

| 概念 | 一句话 |
| :--- | :--- |
| **寄存器** | CPU 内部的超高速工作台，32 个 |
| **栈** | 内存中 LIFO 区域，存局部变量和返回地址 |
| **sp** | 栈指针，指向栈顶，向下增长 |
| **UART** | 向 `0x10000000` 写字符，硬件自动发送 |
| **THRE** | LSR 的 bit5（掩码 `0x20`），1 表示可发送 |
| **`volatile`** | 防止编译器优化，确保硬件访问 |
| **`#ifndef`** | 头文件保护符，防止重复包含 |
| **OpenSBI** | M 模式固件，帮你初始化硬件 |
